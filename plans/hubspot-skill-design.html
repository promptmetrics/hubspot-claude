HubSpot CRM Admin Skill

Version: 2026-05-20 Format: Claude Code Skill (/hubspot) Design

A Claude Code skill that administers HubSpot CRM via natural language. Dispatches 14 specialist sub-agents to perform CRUD, automation, analytics, and data operations. All write operations require human-in-the-loop (HITL) approval.

2. Architecture

Two layers, no custom orchestration framework:

LayerResponsibilityCode location
Parent Orchestrator Parse intent → route to agent(s) → collect results → present preview → HITL approval → execute src/hubspot_agent/orchestrator.py
14 Domain Sub-Agents Each owns a narrow toolset (~4–7 tools) + system prompt. Stateless. Dispatched via Claude Code native Agent tool. src/hubspot_agent/agents/

State management

No custom state framework. Three locations:

  1. Conversation context — parent session carries routing decisions and proposed payloads.
  2. Pending previews — ~/.hubspot-agent/{portal_id}/pending_previews/{action_id}.json
  3. Schema cache — ~/.hubspot-agent/{portal_id}/cache/ (custom objects, properties, pipelines)

Auth

OAuth 2.0 or Private App token only. Portal auto-detected from .hubspot-portal file in working directory. Multi-portal support with isolated state directories.

3. Agent Definitions (14)

1. ObjectsAgent

Core CRM records — contacts, companies, deals, tickets, custom objects

Tools: search, get, create, update, delete, batch_upsert, batch_read

Risk: Medium (create/update/delete). Search is read-only, low risk.

Notes: Custom objects supported via schema cache. Uses object-type-aware property fetching.

2. PropertiesAgent

Schema — custom fields, property groups, validation rules

Tools: list, get, create, update, delete property

Risk: Medium

Notes: Score properties readable but not creatable via API. Calculation properties created via API cannot be edited in HubSpot UI.

3. WorkflowsAgent

Automation — workflows, enrollment triggers, actions

Tools: list, get, create, update, toggle_status, delete

Risk: Medium

Notes: Workflow authoring is complex; agent builds JSON payloads from natural language intent.

4. ListsAgent

Segmentation — static/dynamic lists, memberships

Tools: list, get, create, update, add_members, remove_members

Risk: Medium

5. PipelinesAgent

Pipeline architecture — deal/ticket pipeline stages

Tools: list, get, create, update, reorder_stages

Risk: Medium

Notes: Reordering stages affects all deals in the pipeline — high-impact preview.

6. UsersAgent

User management, owners, teams (read), audit logs

Tools: list_owners, get_user, create_user, delete_user, update_user_profile, list_teams, get_audit_logs, get_login_history

Risk: High (user deletion)

Notes: Team creation and user reassignment to teams are UI-only. Audit logs require Enterprise.

7. HygieneAgent

Data quality — deduplication, merges, bulk updates

Tools: find_duplicates, merge_objects, bulk_update, preview_segment

Risk: High (merge is destructive)

Notes: Dupe detection uses search + fuzzy matching on email/domain/phone.

8. AnalyticsAgent

Reporting — metrics, velocity, funnel calculations

Tools: get_report, calculate_metrics, pipeline_velocity

Risk: Low (read-only)

Notes: Pipeline velocity calculated from deal stage timestamps, not a native API metric.

9. AssociationsAgent

Record relationships — association schemas, links

Tools: get_schema, create_association, delete_association, get_associated

Risk: Medium

10. EngagementsAgent

Activity logging — notes, tasks, emails, meetings, calls

Tools: search, get, create_note, create_task, create_email, create_meeting, create_call

Risk: Medium

11. FormsAgent

Marketing forms — list, create, get, update (caveated), delete

Tools: list, get, create, update, delete

Risk: Medium

Caveat: Forms created in HubSpot’s new web UI editor ("v4 forms") return "BANNED" when updated via API. There is no reliable way to detect v4 vs v3 from the API response. Update tool must document this limitation and suggest UI fallback.

12. DataAgent

Bulk data operations — imports, exports, file manager

Tools: import_csv, get_import_status, export_records, get_export_url, upload_file, list_files

Risk: High (imports can overwrite data)

Notes: Imports require CSV file path + column mapping. Exports are async; returns download URL when ready.

13. CommerceAgent

Commerce objects — line items, products

Tools: search, get, create, update, delete (for both line_items and products)

Risk: Medium

Notes: Line items can be associated with deals and quotes. Products serve as templates for line items.

14. RawAPIAgent

Escape hatch — direct API endpoint access

Tools: hubspot_raw_api

Risk: High

Notes: Bypasses all agent-specific safety rails. HITL approval is mandatory. Scope validation checks OAuth scopes before dispatch.

4. API Constraints & Caveats

ConstraintImpactAgent handling
Forms v4 (new-editor) cannot be updated via API Update fails with "BANNED"; no way to detect beforehand FormsAgent: document caveat, suggest UI fallback on error
Teams cannot be created or reassigned via API User create supports primaryTeamId but only if team already exists UsersAgent: read teams, document UI-only operations
Audit logs are Enterprise-only Non-Enterprise portals get 403 UsersAgent: capability probe before dispatch; graceful decline
Score property rules are in-app only Cannot create or edit scoring criteria via API PropertiesAgent: read score properties, document UI-only
Sequences API is beta, read/enroll only No create/update; no per-sequence email events Out of scope for this revision
Connected apps / integration health has no API Cannot list app connections or sync status Out of scope

5. HITL & Safety

All write operations require human approval. The flow is identical for all 14 agents:

  1. Preview phase — sub-agent returns proposed action + impact count + sample records
  2. Parent presents — "Will update 47 contacts. Approve? (y/n/review)"
  3. Approval modes — Single (one at a time) or Batch (--batch flag)
  4. Timeout reconciliation — If a write times out, parent auto-dispatches a read to verify state
  5. Audit log — Every approved write is logged to ~/.hubspot-agent/{portal_id}/audit/

Risk levels

LevelTriggersPreview requirement
LowSearch, read, list, getOptional summary
MediumCreate, update, toggleImpact count + sample records
HighDelete, merge, bulk update, importFull record list + explicit confirmation

8. Verification Rubric

This skill is done when:

  • Every agent in §3 has a system prompt and working tool bindings
  • All 14 tool modules execute against HubSpot API without unhandled errors
  • Write operations across all agents trigger HITL preview + approval
  • Timeout reconciliation correctly verifies state after write timeouts
  • Audit log writes every approved action to disk
  • /hubspot command parses natural language and routes to the correct agent(s)
  • Multi-portal support works (isolated state, switchable default)
  • All caveats in §4 are documented in agent prompts and user-facing docs
  • 90%+ test coverage on non-UI code
